CREATE TABLE users (id INTEGER PRIMARY KEY AUTOINCREMENT, age INTEGER, name STRING, active BOOLEAN, joined_on DATE); DROP TABLE users; INSERT INTO users (age, name, active, joined_on) VALUES (30, 'Alice', TRUE, DATE '2026-07-23'); INSERT INTO users VALUES (1, 30, 'Alice', TRUE, DATE '2026-07-23'); INSERT INTO users (age, name) VALUES (30, 'Alice'); SELECT * FROM users; SELECT id, name FROM users WHERE age >= 30; SELECT * FROM users WHERE age BETWEEN 30 AND 40; SELECT * FROM users WHERE age = 30 AND name = 'Alice'; SELECT * FROM users WHERE age = 30 OR id = 1; UPDATE users SET name = 'Bob', age = 31 WHERE id = 1; UPDATE users SET age = 31; DELETE FROM users WHERE id = 1; DELETE FROM users; CREATE INDEX users_age_idx ON users (age); DROP INDEX users_age_idx;

Table builder

CREATE TABLE guide

One primary key required
INTEGER

Whole-number values. The current primary key must use this type.

STRING

Quoted text values, for example 'Alice'.

BOOLEAN

Use the literals TRUE or FALSE.

DATE

Use a date literal such as DATE '2026-07-23'.

Keywords PRIMARY KEY AUTOINCREMENT

PRIMARY KEY is required exactly once. AUTOINCREMENT is optional and only valid on an INTEGER PRIMARY KEY column.
